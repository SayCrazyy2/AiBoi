## Go Philosophy

> Go is about **simplicity**, **concurrency**, and **explicitness**.
> - Only 25 keywords
> - No classes, no inheritance (composition instead)
> - Garbage collected
> - Compiled to machine code (blazing fast)
> - Built-in concurrency primitives (goroutines & channels)
> - Strong static typing
